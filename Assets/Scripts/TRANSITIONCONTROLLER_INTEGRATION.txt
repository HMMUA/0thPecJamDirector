TransitionController - SpriteLoader 集成说明
==================================================

【功能概述】

TransitionController 已集成 SpriteLoader，支持从绝对路径加载各类 Sprite：
- 曲绘 (image)
- 标题 (titleImage, titleImageOutline, titleImageMask)
- 谱师 (designerImage, designerImageOutline, designerImageMask)
- 曲师 (artistImage, artistImageOutline, artistImageMask)

特点：
✓ 自动从配置路径加载 Sprite（文件名不需要扩展名）
✓ 智能降级：若绝对路径加载失败，自动转向 Resources
✓ 支持配置化管理
✓ 与 SettingsManager 自动集成


【工作流程】

1. 游戏启动
   ↓
2. SettingsManager 加载配置和应用设置
   ↓
3. TransitionController.ConfigureSpriteLoader() 被调用
   ↓
4. 每当 ResetInformation() 执行时，加载对应的 Sprite
   ↓
5. 使用 LoadSpriteWithFallback() 先尝试 SpriteLoader，再降级到 Resources


【文件命名规则】

基础文件名：{videoId}{suffix}

示例（假设 videoId = "A_001"）：
- 标题：A_001_T（或 A_001_T.png）
- 标题描边：A_001_T_O（或 A_001_T_O.png）
- 谱师：A_001_D（或 A_001_D.png）
- 谱师描边：A_001_D_O（或 A_001_D_O.png）
- 曲师：A_001_A（或 A_001_A.png）
- 曲师描边：A_001_A_O（或 A_001_A_O.png）

支持的扩展名：.png, .jpg, .jpeg, .bmp, .tga


【配置说明】

在 SettingsConfig.json 中配置 SpriteLoader：

{
  "SpriteLoader": {
    "spriteResourcePath": "E:\\0th\\Sprites",
    "useAbsolutePath": true
  }
}

- spriteResourcePath: Sprite 文件夹位置
- useAbsolutePath: 是否使用绝对路径（推荐 true）


【集成步骤】

1. 确保 TransitionController 所在的 GameObject 有 SpriteLoader 组件
   - TransitionController.cs 会自动创建

2. TransitionController 需要接收 VideoData
   - 通过 StartTransition(VideoData) 或 ResetInformation(VideoData)

3. SettingsManager 在启动时自动应用配置
   - FindObjectOfType<TransitionController>() 并调用 ConfigureSpriteLoader()


【方法说明】

【公开方法】

1. ConfigureSpriteLoader(string resourcePath, bool useAbsolute)
   - 手动配置 SpriteLoader 参数
   - 参数：
     * resourcePath: Sprite 文件夹路径
     * useAbsolute: 是否使用绝对路径

   示例：
   ```csharp
   transitionController.ConfigureSpriteLoader("E:\\Sprites", true);
   ```

【私有方法】

2. LoadSpriteWithFallback(string baseName, string suffix)
   - 核心加载方法
   - 先尝试 SpriteLoader，再降级到 Resources
   - 返回加载成功的 Sprite 或 null

3. TryLoadFromResources(string baseName, string suffix)
   - 从 Resources 加载（备选方案）
   - 返回加载成功的 Sprite 或 null


【加载流程】

当 ResetInformation() 执行时：

1. 检验 videoData.id 是否有效
2. 对每个 Sprite 数组执行：
   - 调用 LoadSpriteWithFallback(videoId, suffix)
   - SpriteLoader 尝试加载绝对路径版本
   - 若失败，降级到 Resources.Load
   - 若成功，赋值到相应 SpriteRenderer / SpriteMask


【日志输出】

成功加载（绝对路径）：
  ✓ 标题[0] 加载成功：A_001_T

成功加载（Resources 备选）：
  [TransitionController] 已从 Resources 加载：Texture/A_001_T

加载失败：
  ✗ 标题[0] 加载失败：A_001_T

【打包指南】

【打包前】
1. 创建 Sprites 文件夹，放置所有 Sprite 文件
2. 在 SettingsConfig.json 中设置正确的 spriteResourcePath

【打包后】
1. 编辑 SettingsConfig.json：
   - Windows: 游戏目录/PecJamDirector_Data/Resources/SettingsConfig.json
2. 设置 spriteResourcePath 为实际的 Sprites 文件夹位置
3. 重启游戏


【常见问题】

Q: 为什么有时候用 Resources，有时候用绝对路径？
A: 这是设计的智能降级机制。优先使用绝对路径（更灵活），若失败则自动使用 Resources（更稳定）。

Q: 如果两种方式都失败会怎样？
A: 不会加载任何 Sprite，Debug.Log 会输出警告信息。UI 会保持之前的 Sprite 或空白。

Q: 可以混合使用 Resources 和绝对路径吗？
A: 可以！某些 Sprite 从外部路径加载，某些从 Resources 加载。

Q: 文件名可以不带扩展名吗？
A: 可以！SpriteLoader 会自动尝试 .png, .jpg, .jpeg, .bmp, .tga

Q: 如何查看实际加载的是哪个文件？
A: 查看 Console 日志，会显示详细的加载信息。

Q: 性能会受影响吗？
A: SpriteLoader 内置缓存，相同文件只加载一次，性能无明显影响。


【高级用法】

【示例 1】手动配置 SpriteLoader
```csharp
TransitionController tc = GetComponent<TransitionController>();
tc.ConfigureSpriteLoader("E:\\Games\\Sprites", true);
```

【示例 2】在 SettingsManager 中应用自定义配置
```csharp
SettingsManager.Instance.ReloadSettings();
```

【示例 3】清空 SpriteLoader 缓存
```csharp
SpriteLoader sl = GetComponent<SpriteLoader>();
sl.ClearCache();
```


【与其他系统的集成】

【DirectorSystem】
- DirectorSystem 加载 cover Sprite（封面）到 videoData.coverSprite
- TransitionController 将其用于 image 数组

【SettingsManager】
- 启动时读取 SettingsConfig.json
- 自动调用 TransitionController.ConfigureSpriteLoader()
- 自动调用 SpriteLoader 的配置应用

【VideoData】
- 通过 VideoData.id 获取视频ID
- 自动生成完整的文件名（id + suffix）


【性能优化建议】

1. 使用内置缓存
   - SpriteLoader 默认启用缓存
   - 同一 Sprite 不会重复加载

2. 及时清理缓存
   - 场景切换时调用 ClearCache()
   - 释放不再需要的内存

3. 批量加载
   - 在初始化阶段加载所有需要的 Sprite
   - 避免实时加载导致的卡顿

4. 图片格式选择
   - .png: 256 色以下、透明背景 → 最优
   - .jpg: 全色照片 → 文件最小
   - .bmp/.tga: 特殊用途


【故障排查】

【问题】Sprite 未加载
- 检查日志输出
- 验证文件路径是否正确
- 确认 spriteResourcePath 设置
- 检查文件名大小写（某些系统大小写敏感）

【问题】加载缓慢
- 查看是否有很多 Sprite 在同时加载
- 考虑预加载常用 Sprite
- 检查文件大小和格式

【问题】出现错误信息
- 查看 Console 中的完整错误信息
- 检查 SettingsConfig.json 是否有效
- 重启游戏应用新的配置

【问题】无法从外部路径加载
- 检查路径是否存在
- 检查操作系统权限
- 尝试使用相对路径
- 检查 useAbsolutePath 设置
